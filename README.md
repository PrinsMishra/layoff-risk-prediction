## Quickstart

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Train the model
Open and run all cells in `layoff_risk_training.ipynb`.  
This produces `models/layoff_risk_model.pkl` and `models/model_schema.json`.

### 3. Start the API
```bash
uvicorn app:app --host 0.0.0.0 --port 8000 --reload
```

### 4. Make a prediction
```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "industry": "Software",
    "primary_department": "Engineering",
    "ai_exposure": "Partial",
    "total_employees": 5000,
    "employees_laid_off": 600,
    "severance_weeks": 8,
    "layoff_reason": "Profitability challenges",
    "month": 1,
    "quarter": 1
  }'
```

**Example response:**
```json
{
  "request_id": "a3f7...",
  "timestamp": "2026-02-16T00:00:00Z",
  "risk_label": "HIGH",
  "risk_probability": 0.8421,
  "risk_score": 84,
  "percentage_workforce_affected": 12.0,
  "top_risk_factors": [
    "Large workforce reduction (12.0% of headcount)",
    "Q1 — historically the highest layoff quarter"
  ],
  "model_version": "XGBoost",
  "latency_ms": 12.4
}
```

## ELK Stack / Kibana Integration

All prediction requests are logged to stdout in structured JSON, e.g.:

```json
{
  "timestamp": "2026-02-16T01:00:00Z",
  "level": "INFO",
  "message": "prediction",
  "industry": "Fintech",
  "department": "Engineering",
  "risk_label": "HIGH",
  "risk_probability": 0.82,
  "risk_score": 82,
  "latency_ms": 11.2
}
```

Configure Logstash to read from the container's stdout and index into Elasticsearch.  
Recommended Kibana dashboards:
- **Industries by Query Volume** — terms agg on `industry`
- **Average Risk Score Over Time** — date histogram on `timestamp`, avg of `risk_score`
- **Risk Label Distribution** — pie chart on `risk_label`
- **API Latency** — percentile agg on `latency_ms`

## Continuous Learning

When new layoff data arrives (LinkedIn scraper, news feed), drop the CSV in `data/`
and call `retrain_on_new_data()` from the notebook (or trigger it via Airflow/cron):

```python
retrain_on_new_data("data/new_layoffs_march_2026.csv")
```

The champion-challenger check ensures the model is only replaced if the new AUC ≥ current AUC.
